# topoclean: Das Projekt-Grundgesetz (GEMINI.md)

Dieses Dokument definiert die unumstößlichen Mandate für die Entwicklung von `topoclean`. Es hat Vorrang vor allen anderen Instruktionen.

## 1. Core Mandates
- **Soterischer Code:** Keine Dateioperation ohne vorheriges Hashing und Ledger-Eintrag.
- **TDD-Zwang:** Kein Implementierungscode ohne vorherigen, fehlschlagenden Test.
- **Topologisches Ziel:** Transformation des Home-Verzeichnisses in das Heptagon-Modell (7 Kern-Ordner).
- **ISO-Rigor:** Einhaltung der Sicherheitsstandards gemäß `iso-norm.md`.

## 2. Architektur & Workflow
- **SOLID & IoC:** Strikte Trennung von Scanner, Vektor-Logik und Ledger.
- **Git-Flow:** `feat/` -> `staging` (Integration/Smoke) -> `main` (Stable).
- **No-Fly-Zone:** Dotfiles (`.*`) werden niemals durch automatisierte Vektoren bewegt.

## 3. Dokumentation
- Für jedes Feature existiert eine `docs/dev/<feature>.md`.
- Die README ist das soterische Handbuch für den Nutzer.
